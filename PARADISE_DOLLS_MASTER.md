### Key Module: Premium vs Freemium Websites
This module should be implemented in the LMS. Content from May 8, 2026 lesson email:

**Lesson 1 — Understanding Freemium / Token Websites**
- Public chat-based platforms, viewers enter free
- Token/credit tipping system
- Goal: entertain, attract attention, encourage tipping, upsell privates
- Examples: Chaturbate, Stripchat, BongaCams, Cam4, CamSoda
- Benefits: massive traffic, fan building, go viral potential
- Downsides: freeloaders, energy-intensive, lower avg spend

**Lesson 2 — Understanding Premium Websites**
- Pay-per-minute for private access
- Examples: AdultWork, Babestation, Xpanded/XXXpanded, Streamate
- Benefits: higher quality spenders, direct earnings, fewer viewers but more serious
- Downsides: less traffic, slower fan growth

**Lesson 3 — Premium vs Freemium & How the Industry is Changing**
- Premium sites now adopting freemium features (tipping, interactive rooms)
- Smart creators combine both: freemium for visibility → premium for earnings
- Strategy called multi-streaming
- Paradise Dolls teaches BOTH sectors

## 4. Model Onboarding Flow (8 Steps)

### Step 1 — Model submits application form
- Goes to website, completes and submits application form
- **Result**: Admin email notified, submission appears in admin dashboard

### Step 2 — Admin reviews application
- Dashboard → Form Submissions
- Checks: age (18+), photos
- **If approved**: Send **Application Approval Email**

### Step 3 — Model receives Application Approval Email
- Model must complete TWO things:
  1. Fill in **Model Information Form** (Platform Information Form)
  2. Create **Member Account Login**
- **Result**: Form 2 submission appears, new member request appears

### Step 4 — Admin checks progress
- Confirm Model Information Form submitted
- Confirm Member Account created
- **If both complete**: Approve member → Send **Verification Request Email**

### Step 5 — Model completes verification (Form 3)
- Model logs into account
- Accesses verification page (members only)
- Uploads: Valid ID + Selfie holding ID / Platform Codes
- **Result**: Verification form submission appears

### Step 6 — Admin reviews verification
- Check ID valid, face matches, confirm age
- **If approved**: Mark model as verified → Send **Account Approval Email**

### Step 7 — Invite to community
- Send **Community Access Email**
- Include invite link to Discord server
- Assign role: "Boss Doll Blueprint"

### Step 8 — Model fully onboarded
- Completed all forms
- Verified identity
- Member account access
- Joined community
- Ready for: platform setup, training, going live

### Email Templates (6 total)
1. Application Approval Email
2. Model Information Confirmation Email
3. Verification Request Email
4. Verification Submission Confirmation Email
5. Account Approval Email
6. Community Access Email

## 5. Technical Architecture

### Stack
- **Framework**: Laravel 11 (PHP)
- **Frontend**: Blade templates + Alpine.js
- **Styling**: Tailwind CSS v3 + custom `boss-*` token system
- **Build tool**: Vite
- **Video hosting**: Bunny Stream (BunnyStreamClient service)
- **Database**: MySQL

### App Structure
```
app/
  Http/Controllers/
    ApplyController          — public application form
    HomeController           — homepage with testimonials
    TestimonialController    — success stories page
    Admin/
      AdminApplicationController   — review/approve/reject applications
      AdminCourseController        — course CRUD
      AdminLessonController        — lesson CRUD
      AdminDashboardController     — admin overview stats
      AdminModelProgressController — model progress table
      AdminOnboardingController    — verification & community flow
      AdminTestimonialController   — testimonials CRUD
      BunnyVideoController         — Bunny video upload intent
    Member/
      MemberDashboardController    — member home with progress
      MemberCourseController       — courses listing, show, learn
      LessonProgressController     — mark lesson complete
      MemberOnboardingController   — platform info form
      MemberVerificationController — ID upload form
      CourseChatController         — per-course community chat
  Models/
    User                 — role: admin / model
    ModelApplication     — application submissions (status: pending/approved/rejected)
    ModelProfile         — verification state, onboarding fields, community status
    Course               — LMS courses (slug, title, published, Bunny intro video)
    Lesson               — lessons per course (Bunny video, PDF, Canva embed)
    LessonProgress       — per-user lesson completion
    CourseEnrollment     — per-user course access
    ChatRoom             — per-course community room
    CourseChatMessage    — chat messages
    Testimonial          — success stories (image, quote, headline, visible flag)
  Mail/                  — 6 email mailable classes
  Services/
    BunnyStreamClient    — Bunny Stream API integration
```

## 6. Deployment — Hostinger Guide

### Pre-deployment Checklist

**Environment file (`.env` production values)**
```env
APP_ENV=production
APP_DEBUG=false
APP_URL=https://yourdomain.com

DB_CONNECTION=mysql
DB_HOST=your-hostinger-db-host
DB_PORT=3306
DB_DATABASE=your_db_name
DB_USERNAME=your_db_user
DB_PASSWORD=your_db_password

SESSION_DRIVER=database
CACHE_STORE=database
QUEUE_CONNECTION=database

MAIL_MAILER=smtp
MAIL_HOST=smtp.hostinger.com
MAIL_PORT=465
MAIL_USERNAME=admin@getrichwithparadisedolls.com
MAIL_PASSWORD=your-email-password
MAIL_ENCRYPTION=ssl
MAIL_FROM_ADDRESS="admin@getrichwithparadisedolls.com"
MAIL_FROM_NAME="Paradise Dolls"

FILESYSTEM_DISK=local
```

**File storage for ID documents** (sensitive — must stay private):
- Store at `storage/app/private/` (outside public web root)
- Never use `public` disk for verification documents

**Build steps on server:**
```bash
composer install --no-dev --optimize-autoloader
php artisan key:generate --force
php artisan migrate --force
php artisan config:cache
php artisan route:cache
php artisan view:cache
npm ci && npm run build
```

**Hostinger-specific:**
- Document root must point to `/public`
- PHP version: 8.2+
- Enable `mod_rewrite` (`.htaccess` is already in `public/`)
- Set `upload_max_filesize` and `post_max_size` to at least 20MB (for ID uploads and photos)
- Cron for queue (if using database queue): `* * * * * php /path/to/artisan queue:work --stop-when-empty`

**Storage symlink:**
```bash
php artisan storage:link
```

**Permissions:**
- `storage/` and `bootstrap/cache/` must be writable (755 or 775)
